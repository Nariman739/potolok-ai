# Mobile Spike — Этап 3.5 плана cryptic-yawning-raccoon

## Зачем

Перед началом Этапа 4 (mobile-вход через вариант C: 2D-snapshot плана + RoomElement[] JSON) проверяем что Gemini 2.5 Flash Image справляется с **TOP-DOWN 2D планом + фото комнаты** и выдаёт картинку, которую не стыдно показать клиенту на закрытии сделки.

Если на спайке провал — переходим на серверный 3D-рендер (вариант B/A), +3-5 дней к Этапу 4.

## Как готовить пары

В корне проекта создать папку `spike-pairs/`:

```
spike-pairs/
├── pair1/
│   ├── photo.jpg     # Фото реальной комнаты (любой ракурс с любым потолком)
│   ├── plan.png      # 2D-план сверху из конструктора (со всеми элементами)
│   └── meta.json     # { "finish": "matte", "colorName": "белый" }
├── pair2/
│   ├── photo.jpg
│   ├── plan.png
│   └── meta.json
└── ...
```

**Сколько пар нужно:**
- 3-5 пар Наримана (исторические объекты)
- 1-2 пары Серика (свежий замер на ближайшем выезде — для контроля false positive)

**meta.json формат:**
```json
{
  "finish": "matte",          // matte | satin | glossy
  "colorName": "белый",       // или "чёрный", "карамель" и т.д.
  "extraPrompt": ""           // опционально, доп. инструкции в промпт
}
```

## Запуск

```bash
cd ~/projects/potolok-ai
npx tsx scripts/spike-mobile-quality.ts ./spike-pairs/
```

Скрипт обработает все пары последовательно. Результаты — в `spike-results/`:
```
spike-results/
├── pair1_2026-05-25T...png        # AI-рендер
├── pair1_2026-05-25T....prompt.txt # промпт для отладки
├── pair1_2026-05-25T....meta.json  # время рендера + параметры
└── ...
```

## Критерий приёмки

Нариман смотрит ВСЕ результаты глазами. Критерий:

> «Картинку не стыдно показать клиенту на закрытии сделки»

Если на ВСЕХ парах ОК (свои + Серика) → Этап 4 идём по варианту C (mobile отправляет 2D + RoomElement JSON, без 3D на устройстве).

Если на исторических ОК, но на свежем замере Серика — провал → false positive выявлен. Переходим на серверный 3D-рендер (mobile отправляет только RoomElement JSON, сервер строит сцену через three.js).

Если плохо везде → то же решение, серверный 3D.

## Не забыть

- `spike-pairs/` и `spike-results/` в .gitignore (личные данные клиентов).
- `OPENROUTER_API_KEY` должен быть в `.env.local`.
- Стоимость одного прогона: ~$0.04 (Gemini 2.5 Flash Image, OpenRouter).
- Стоимость всего спайка (5-7 пар): ~$0.30. Бюджет совсем небольшой.
